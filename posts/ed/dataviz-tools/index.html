Освіта: Як обрати інструмент для візуалізації даних?

2017-06-06

Німецька журналістка Ліза-Шарлотта Рост провела цікавий експеримент з візуалізації даних - вона спробувала відтворити один й той самий графік за допомоги 24 різних інструментів (12 програм чи застосунків та 12 бібліотек для візуалізації даних). Про результати експерименту вона написала у замітці What I Learned Recreating One Chart Using 24 Tools.

Головний висновок експерименту - немає ідеальних інструментів для візуалізації даних загалом, але є хороші інструменти для певних задач у сфері візуалізації даних. Отже, вибираючи інструмент для візуалізації даних, слід чітко усвідомлювати, які задачі ви намагаєтесь вирішити.

Зокрема слід зрозуміти, для чого вам візуалізація даних - для аналізу чи презентації? Візуалізація потрібна вам для себе, для дослідження даних, вирізнення в них тренду чи паттерну? Чи ви хочете створити щось для певної аудиторії?

Чи потрібно вам робити якісь операції із даними під час візуалізації? Тому що деякі інструменти чи бібліотеки для візуалізації даних дозволяють обчислювати нові змінні чи агрегувати дані на льоту, а інші вимагають вже максимально підготовлених для візуалізації даних. Тобто вам доведеться зробити всі обчислення деінде і вже імпортувати до програми підготовлені дані.

Вам достатньо типових графіків чи ви хочете щось дуже нестандартне, специфічне? Деякі інструменти дозволяють дуже швидко зробити типові візуалізації (стовпчикові діаграми, лінійні графіки, діаграми розсіяння і т.д.), але найдрібніша кастомізація при цьому займе у вас дуже багато часу. Інші потребують трохи більше часу навіть для стандартних графіків, але при цьому дуже гнучкі та дозволяють робити дуже нетипові штуки.

Вам достатньо статичної графіки чи обов’язково мати інтерактив? Відповідаючи на це питання, зважайте на нещодавню дискусію щодо доцільності використання інтерактивності у візуалізаціях. Не ховайте важливі дані, не змушуйте користувачів клікати чи наводити курсор на візуальні елементи, аби отримати справді важливі дані, використовуйте анотації, анімацію та transitions.
